# 网站结构设计(信息架构 IA)

> 来源:由 `keyword-matrix.csv` 推导。原则——**每一类页面都对应矩阵里的一类词,没有目标词的页面不建。**

## 一、站点树

```
首页 /  (工具首页 · 核心词「明日方舟人物关系图」)
│
├── 人物名录 /characters/                (分类页 · 「明日方舟人物/干员」)
│   └── 人物百科 /characters/{id}/        (内容页 · 实体词+「X是谁」长尾) ×11
│
├── 阵营总览 /factions/                  (分类页 · 「明日方舟势力」)
│   └── 阵营专题 /factions/{id}/          (子分类页 · 「某阵营成员有哪些」) ×4
│
├── 关系总览 /relations/                 (专题入口)
│   └── 关系对页 /relations/{a}-{b}/      (FAQ集合 · 「X和Y什么关系」长尾) ×13
│
├── SEO 笔记 /seo-notes/                 (博客页 · 学习记录)
│
└── 信任/合规
    ├── 关于本站 /about/                 (E-E-A-T 信任信号)
    └── 隐私政策 /privacy/
```

## 四、架构原则

1. **扁平化**:任意页面距首页 ≤ 3 次点击(首页→阵营→人物 / 首页→人物→关系)。
2. **权重向上汇聚**:长尾(关系对页)→ 专题(阵营页)→ 核心(首页),用内链把权重一级级顶上去。
3. **三向内链**:人物页 ↔ 关系对页 ↔ 阵营页 互相链接,关系图节点也链到人物页。
4. **面包屑回流**:每个内容页带 `首页 / 分类 / 当前` 面包屑,既导航又传权重。
5. **URL 规范**:全小写、英文语义化 id、统一带尾斜杠(astro.config 已配 trailingSlash)。
6. **结构化数据按页型分配**:见上表「结构化数据」列——这是 GEO/AI 引用的基础。
